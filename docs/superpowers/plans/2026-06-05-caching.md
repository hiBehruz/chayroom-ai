# Caching Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add the approved three-layer cache: Redis-backed public API caching, SWR for safe public pages, and Caddy immutable static asset headers.

**Architecture:** Nitro cached handlers will cache only public GET responses under explicit names and keys. Admin mutations will clear related cached entries through a small server utility. Public route SWR and Caddy static headers will be configured conservatively to avoid per-user SSR leaks.

**Tech Stack:** Nuxt 4, Nitro cache helpers, unstorage Redis driver, Docker Compose, Caddy.
